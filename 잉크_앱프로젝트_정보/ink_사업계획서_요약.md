# [사업계획서] ink – 요약

**실시간 벡터 동기화 기반 전문 협업 무소음 메신저: "ink"**

---

## 1. 프로젝트 개요 (Executive Summary)

| 항목 | 내용 |
|------|------|
| **서비스명** | ink (잉크) |
| **핵심 가치** | "가장 조용한 곳에서, 가장 복잡한 생각을 실시간으로 잇다" |
| **제품 정의** | 태블릿의 펜 스트로크(Stroke)를 실시간 벡터 좌표로 전송하여, 수식·도면·디자인 시안을 대화하듯 수정·공유하는 하이브리드 무소음 커뮤니케이션 플랫폼. |

---

## 2. 문제 인식 및 개발 배경 (Problem & Necessity)

### 2-1. 시장의 페인 포인트 (Pain Points)

1. **공간적 제약과 소음 갈등**  
   도서관, 강의실, 스터디카페 등 'Silent Zone'에서 메신저 사용 시 키보드 타건 소음으로 인한 이용자 간 분쟁 급증.

2. **비정형 데이터 전달의 불가능**  
   수식, 화학식, 설계 도면, 아이디어 스케치 등은 기존 텍스트 메신저(카카오톡, 슬랙 등)로 설명 시 정보의 왜곡과 소통 비용(Time-cost)이 과다 발생.

3. **정지된 이미지 기반 소통의 한계**  
   기존 앱은 '이미지 캡처 후 전송' 방식으로, 상대방이 어떤 순서로 사고하고 수정하는지 실시간 추적이 불가능함.

### 2-2. [강력한 근거] 데이터로 보는 시장 기회

- **태블릿 보급률:** 국내 대학생 80% 이상이 태블릿 보유. 2024년 기준 대학 신입생의 92%가 태블릿을 학습 필수품으로 인식.
- **행동 패턴의 변화:** 종이 대신 PDF와 스타일러스 펜을 사용하는 '페이퍼리스' 학습자가 85%에 육박하지만, 이들을 위한 실시간 필기 채팅 툴은 전무한 상태.

## 3. 해결 방안 및 핵심 기능 (Solution & Features)

### 3-1. 핵심 기술: 실시간 벡터 스트리밍 (Ghost Writing)

- **기술적 구현:** 이미지 파일 전송이 아닌, 펜의 (x, y) 좌표, 압력(Pressure), 타임스탬프를 최소 단위의 JSON 데이터로 스트리밍.
- **사용자 경험:** 상대방이 선을 긋는 궤적이 내 화면에 0.1초 미만의 지연 시간으로 실시간 렌더링되어 현장감 극대화.
